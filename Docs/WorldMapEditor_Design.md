# World Map Editor — Проектная документация

## Обзор

Браузерное веб-приложение для создания фэнтези-карт мира. Предназначено для worldbuilding — создания карт для книг и настольных ролевых игр. Персональный инструмент без бэкенда.

Все объекты на карте представлены кривыми (сплайнами). Данные хранятся в JSON-файле на диске пользователя. SVG-рендер генерируется из данных с использованием seed-based алгоритмов — внешний вид воспроизводим, но "живой".

---

## Стек технологий

- **Фронтенд:** HTML + JavaScript (vanilla, без фреймворков)
- **Рендеринг:** SVG
- **Хранение данных:** JSON-файл, загрузка/сохранение через File System Access API (или fallback на download/upload)
- **Бэкенд:** отсутствует
- **Сервер:** отсутствует — приложение запускается двойным кликом по `index.html`

### Целевые платформы

- Chrome на Windows
- Safari на macOS

## Структура проекта

Приложение распределено по файлам для удобства поддержки, но не требует сборки или сервера. Все файлы загружаются браузером напрямую по `file://` через обычные теги `<script>`.

```
WorldMapEditor/
├── index.html
├── css/
│   └── style.css
├── libs/
│   └── simplex-noise.js       ← сторонняя библиотека, хранится локально
└── js/
    ├── app.js                 ← точка входа, инициализация
    ├── state.js               ← глобальное состояние карты (App.state)
    ├── spline.js              ← работа с кривыми Безье
    ├── noise.js               ← fractal displacement поверх simplex
    ├── renderer.js            ← SVG-рендер всех объектов
    ├── editor.js              ← логика редактора (режимы, выделение)
    ├── ui.js                  ← панели, список объектов, свойства
    └── storage.js             ← сохранение/загрузка JSON, экспорт SVG
```

Подключение в `index.html`:

```html
<link rel="stylesheet" href="css/style.css">

<script src="libs/simplex-noise.js"></script>
<script src="js/state.js"></script>
<script src="js/spline.js"></script>
<script src="js/noise.js"></script>
<script src="js/renderer.js"></script>
<script src="js/editor.js"></script>
<script src="js/storage.js"></script>
<script src="js/ui.js"></script>
<script src="js/app.js"></script>
```

### Соглашения по коду

Поскольку ES-модули (`import`/`export`) не работают по `file://`, весь код организован через единый глобальный namespace `App`:

```javascript
// state.js
var App = App || {};
App.state = { ... };

// renderer.js
var App = App || {};
App.renderer = { ... };
```

Каждый файл объявляет свою часть `App` в начале. Порядок подключения `<script>` в `index.html` определяет порядок инициализации.

## Модель данных (JSON)

### Корневая структура

```json
{
  "version": 1,
  "name": "Мир Кланов",
  "seed": 42,
  "viewport": { "x": 0, "y": 0, "zoom": 1.0 },
  "objects": []
}
```

### Объект карты

Каждый объект — кривая (замкнутая или незамкнутая) или точка-маркер.

```json
{
  "id": "uuid-string",
  "type": "region",
  "name": "Вороны",
  "showLabel": true,
  "closed": true,
  "points": [
    { "x": 100, "y": 50, "cx1": 90, "cy1": 45, "cx2": 110, "cy2": 55 }
  ],
  "seed": 17,
  "style": {},
  "params": {}
}
```

Поле `points` — массив контрольных точек кубического сплайна. Каждая точка содержит позицию (`x`, `y`) и два управляющих маркера (`cx1/cy1`, `cx2/cy2`) для входящей и исходящей касательных.

## Типы объектов

### Замкнутые кривые

#### `region` — Политический регион / страна

Основной тип. Замкнутая область с заливкой.

| Параметр | Тип | Описание |
|---|---|---|
| `fillColor` | string | Цвет заливки |
| `borderNoise.frequency` | number | Частота излома границ |
| `borderNoise.amplitude` | number | Амплитуда излома |
| `borderNoise.octaves` | number | Количество октав шума (детализация) |

#### `sea` — Море / океан

Замкнутая водная область большого масштаба.

| Параметр | Тип | Описание |
|---|---|---|
| `fillColor` | string | Цвет воды (по умолчанию синий) |
| `wavePattern` | boolean | Рисовать ли волновой паттерн |
| `borderNoise.frequency` | number | Частота излома береговой линии |
| `borderNoise.amplitude` | number | Амплитуда излома |

#### `lake` — Озеро

Аналогично морю, но меньшего масштаба.

| Параметр | Тип | Описание |
|---|---|---|
| `fillColor` | string | Цвет воды |
| `borderNoise.frequency` | number | Частота излома берегов |
| `borderNoise.amplitude` | number | Амплитуда излома |

#### `biome` — Биом / природная зона

Замкнутая область для обозначения природных зон: тундра, тайга, болото, лес, равнина, фьорды.

| Параметр | Тип | Описание |
|---|---|---|
| `biomeType` | string | Тип: `tundra`, `taiga`, `swamp`, `forest`, `plains`, `fjord` |
| `fillPattern` | string | Паттерн заливки (зависит от `biomeType`) |
| `density` | number | Плотность паттерна (напр. частота деревьев) |
| `opacity` | number | Прозрачность слоя биома |

### Незамкнутые кривые

#### `river` — Река

Незамкнутая кривая, рендерится как линия переменной толщины.

| Параметр | Тип | Описание |
|---|---|---|
| `widthStart` | number | Толщина в начале (исток) |
| `widthEnd` | number | Толщина в конце (устье) |
| `sinuosity` | number | Извилистость (амплитуда волн) |
| `color` | string | Цвет воды |

#### `mountain` — Горный хребет

Незамкнутая кривая, вдоль которой генерируются горные пики.

| Параметр | Тип | Описание |
|---|---|---|
| `width` | number | Ширина зоны гор от центральной кривой |
| `density` | number | Плотность пиков |
| `heightVariation` | number | Вариация высоты пиков |
| `fadeStart` | number | Затухание в начале (0..1) |
| `fadeEnd` | number | Затухание в конце (0..1) |

### Точки-маркеры

#### `marker` — Город / точка интереса

Одиночная точка на карте.

| Параметр | Тип | Описание |
|---|---|---|
| `markerType` | string | Тип: `city`, `capital`, `ruin`, `landmark`, `port` |
| `iconSize` | number | Размер иконки |

## Алгоритм рендеринга границ

### Задача

Превратить гладкие кривые Безье из JSON в "живые" изломанные границы, характерные для фэнтези-карт.

### Алгоритм: Fractal Displacement

1. **Сэмплирование кривой.** Кубический сплайн разбивается на N равноудалённых точек (шаг зависит от масштаба).

2. **Фрактальное смещение.** Для каждой точки вычисляется нормаль к кривой. Точка смещается вдоль нормали на величину, определяемую Simplex-шумом:

   ```
   offset = simplex(x * frequency, y * frequency, seed) * amplitude
   ```

   Применяется несколько октав — крупные "заломы" + мелкая детализация:

   ```
   total = 0
   for i in 0..octaves:
       total += simplex(x * freq * 2^i, y * freq * 2^i, seed + i) * amp / 2^i
   ```

3. **Seed-воспроизводимость.** Каждый объект имеет свой `seed`. При одинаковых данных и seed — идентичный результат.

### Снаппинг границ соседних регионов

Выполняется **на этапе рендера**, исходные кривые в JSON не изменяются.

1. **Поиск соседей.** Для каждой пары регионов проверяется пересечение bounding box.
2. **Зона притяжения.** Если расстояние между сэмплированными точками двух границ < `snapThreshold`, точки сливаются в среднюю позицию.
3. **Общая граница.** Участок, где два региона "слиплись", получает одну общую ломаную линию (вычисленную один раз и используемую обоими регионами).
4. **Приоритет.** Если регион A и регион B имеют разные параметры шума — используется среднее значение параметров для общего участка.

### Заполнение пустот

Если между регионами остаётся зазор шире `snapThreshold` — он остаётся (это может быть нейтральная территория). Пользователь контролирует `snapThreshold` глобально.

## Рендеринг объектов

### Регионы / биомы

- Замкнутые SVG `<path>` с изломанными границами
- Заливка `fill` сплошным цветом или SVG `<pattern>` для биомов
- Подпись `<text>` по центру области (если `showLabel: true`)

### Моря и озёра

- Замкнутые `<path>` с заливкой цветом воды
- Опциональный волновой паттерн через SVG `<pattern>` с горизонтальными волнистыми линиями

### Реки

- SVG `<path>` с `stroke-width`, интерполируемым от `widthStart` до `widthEnd`
- Дополнительная извилистость через синусоидальное смещение точек с параметром `sinuosity`

### Горные хребты

- Вдоль кривой с шагом `1 / density` размещаются SVG-символы гор (треугольники с вариацией)
- Позиция каждого символа: точка на кривой + случайное смещение (seed-based) в пределах `width`
- Высота символа: базовая * `heightVariation` * noise
- Затухание (`fadeStart`, `fadeEnd`): масштаб символов плавно уменьшается к краям кривой

### Маркеры

- SVG `<use>` со ссылкой на предопределённый символ (`city`, `capital`, `ruin`, `landmark`, `port`)
- Подпись рядом с маркером (если `showLabel: true`)

### Левая панель — Список объектов

- Древовидная структура, сгруппированная по типам
- Drag & drop для изменения порядка слоёв
- Иконки видимости (глаз) для каждого объекта
- Клик — выделение объекта на canvas и открытие его свойств

### Центральная область — Canvas

- SVG viewport с pan (средняя кнопка / пробел + drag) и zoom (колёсико)
- Режим Select: клик по объекту выделяет, показывает контрольные точки
- Режим Draw: клик ставит точки, двойной клик завершает кривую
- Редактирование точек: drag для перемещения, drag управляющих маркеров для изменения кривизны
- Delete/Backspace — удаление выбранных точек
- Ctrl+клик по сегменту — вставка новой точки

### Правая панель — Свойства

- Отображает параметры выбранного объекта
- Поля зависят от типа объекта
- Изменения применяются мгновенно (re-render)

## Сохранение и экспорт

### JSON (рабочий формат)

Полное состояние карты. Сохранение и загрузка реализуются через File System Access API — он поддерживается в Chrome и Safari при открытии файла по `file://`.

```javascript
// Сохранение
const handle = await window.showSaveFilePicker({
  types: [{ description: 'Map JSON', accept: { 'application/json': ['.json'] } }]
});
const writable = await handle.createWritable();
await writable.write(JSON.stringify(mapData, null, 2));
await writable.close();

// Загрузка
const [handle] = await window.showOpenFilePicker({
  types: [{ description: 'Map JSON', accept: { 'application/json': ['.json'] } }]
});
const file = await handle.getFile();
const mapData = JSON.parse(await file.text());
```

Fallback для браузеров без поддержки File System Access API: сохранение через download blob, загрузка через `<input type="file">`.

### SVG (экспорт)

Финальная карта экспортируется как SVG-файл. Это чистый рендер без элементов редактора (контрольные точки, сетка, выделение).

## Что НЕ входит в проект

- Бэкенд / серверная часть
- Локальный сервер / сборщик (webpack, vite и т.п.)
- Аккаунты и авторизация
- Совместное редактирование
- Отдельный инструмент текста (подписи — через поле name)
- Real-time снаппинг границ (только при рендере)
- AI/нейросетевая генерация (возможно в будущем)
- Мобильная версия
